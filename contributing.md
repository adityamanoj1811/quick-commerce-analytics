# Contributing to Quick-Commerce Delivery Analytics

Thank you for your interest in contributing! Here are the guidelines to follow.

---

## 🛠️ How to Contribute

### 1. Fork the Repository
Click the **Fork** button at the top right of this page to create your own copy.

### 2. Clone Your Fork
```bash
git clone https://github.com/YOUR_USERNAME/quick-commerce-analytics.git
cd quick-commerce-analytics
```

### 3. Create a New Branch
```bash
git checkout -b feature/your-feature-name
```

### 4. Make Your Changes
- Keep code clean and well-commented
- Follow existing notebook section structure
- Test your code before submitting

### 5. Commit and Push
```bash
git add .
git commit -m "Add: brief description of your change"
git push origin feature/your-feature-name
```

### 6. Open a Pull Request
Go to the original repo and click **New Pull Request**.

---

## 📋 Contribution Ideas

- Add new visualisation types (interactive Plotly charts)
- Improve ML model hyperparameter tuning
- Add SARIMA / Prophet time series models
- Add association rules analysis
- Improve README documentation

---

## 📐 Code Style

- Use clear variable names
- Add comments for each major step
- Keep functions short and focused
- Use markdown cells to explain analysis
